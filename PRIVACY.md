# ValidationForge Privacy Policy

ValidationForge includes **opt-in** usage telemetry to help the maintainers understand how the tool is being used and prioritize future improvements. Telemetry is **disabled by default** and must be explicitly enabled by the user.

> **TL;DR:** No PII is ever collected. You choose to opt-in. You can disable at any time. A random anonymous ID is generated â€” it has no connection to you.

## Data Collected

When telemetry is enabled, the following anonymized metrics are collected:

| Field | Example | Purpose |
|-------|---------|---------|
| `event` | `command.invoked` | Which event occurred |
| `command` | `validate` | Which VF command was invoked |
| `platform` | `web` | Platform type detected (web, api, ios, cli, fullstack) |
| `phase` | `execute` | Pipeline phase completed (plan, preflight, execute, etc.) |
| `verdict` | `pass` | Validation outcome pass or fail (pass/fail ratio tracking) |
| `vf_version` | `1.2.0` | ValidationForge version in use |
| `anonymousId` | `a3f7c2d1-...` | Random UUID generated at opt-in time â€” no link to identity |
| `timestamp` | `2026-04-10T19:00:00Z` | UTC timestamp of the event |

### Events Emitted

| Event Name | Emitted When |
|------------|-------------|
| `command.invoked` | A VF command starts executing |
| `platform.detected` | Auto-detection identifies the project platform |
| `pipeline.phase.completed` | A pipeline phase (plan/preflight/execute/analyze/verdict) completes |
| `validation.verdict` | Final pass/fail verdict is produced |

## Anonymous ID

When you opt-in to telemetry, a random UUID is generated and stored in `~/.claude/.vf-config.json` as `telemetry.anonymousId`. This ID:

- Is generated using `uuidgen` (macOS/Linux) or `python3 -c "import uuid; print(uuid.uuid4())"`
- Has **no connection** to your identity, username, system, or project
- Is used solely to count distinct installations for adoption metrics
- Can be regenerated by disabling and re-enabling telemetry
- Is never shared with third parties

## Telemetry Endpoint

Events are sent via HTTPS POST to:

```
https://telemetry.validationforge.dev/events
```

Only HTTPS endpoints are permitted â€” the telemetry script validates the URL scheme and rejects non-HTTPS targets. All transmission is fire-and-forget with a 5-second timeout (`--max-time 5`). Failures are silent and never block VF operations.

## How to Enable Telemetry

Telemetry is **opt-in** and disabled by default. To enable:

```bash
vf telemetry enable
```

This will:
1. Display a disclosure listing every field that will be collected
2. Generate a random `anonymousId` UUID
3. Update `~/.claude/.vf-config.json` with `telemetry.enabled: true`
4. Confirm activation

During `/vf-setup`, you will also be prompted with an opt-in question and can enable telemetry as part of first-time setup.

## How to Disable Telemetry

To disable telemetry at any time:

```bash
vf telemetry disable
```

This sets `telemetry.enabled: false` in `~/.claude/.vf-config.json`. No further data is collected after this point. Your `anonymousId` is retained in config (it is not sensitive) but no events are sent.

## View the Exact Payload

To see precisely what data would be sent before enabling, or to audit the payload at any time:

```bash
vf telemetry show
```

This prints the exact JSON object that would be transmitted to the endpoint for a sample `command.invoked` event. No data is sent when running this command.

## Check Telemetry Status

```bash
vf telemetry status
```

Shows whether telemetry is enabled or disabled, and displays the first 8 characters of your `anonymousId` (sufficient to confirm identity without exposing the full UUID).

## Data Retention

Telemetry events are retained for **90 days** on the server side, after which they are permanently deleted. Aggregate summaries (e.g., "N users invoked /validate this month") may be retained indefinitely, but these contain no per-installation data and cannot be traced back to any individual or installation.

## Configuration Storage

Telemetry configuration is stored locally in `~/.claude/.vf-config.json`:

```json
{
  "telemetry": {
    "enabled": false,
    "anonymousId": "",
    "endpoint": "https://telemetry.validationforge.dev/events"
  }
}
```

- **`enabled`**: `false` by default. Must be explicitly set to `true` via opt-in.
- **`anonymousId`**: Empty until opt-in; populated with a random UUID at opt-in time.
- **`endpoint`**: The HTTPS telemetry endpoint. Only HTTPS URLs are accepted.
