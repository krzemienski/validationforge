ValidationForge ships **41 skills** across **7 categories**. Every skill
uses Claude Code `SKILL.md` frontmatter so Claude Code (and OpenCode)
can auto-discover it at session start. Ten of the most important skills
have full documentation pages on this site; the remaining 31 link out
to their `SKILL.md` source in the GitHub repository.

<Aside type="tip" title="Ten skills have detail pages">
  Skills in **bold with a linked name** have a full documentation page
  on this site. The rest link to their `SKILL.md` in the GitHub repo —
  the same file Claude Code reads at discovery time, so the in-repo
  source is the canonical reference.
</Aside>

## Why Only Ten Detail Pages?

The ten skills with dedicated pages were chosen because they cover the
ValidationForge story end-to-end: the three Iron-Rule quality gates
(`functional-validation`, `gate-validation-discipline`,
`no-mocking-validation-gates`), the pre-flight discipline (`preflight`),
every supported platform (`web-validation`, `api-validation`,
`ios-validation`, `playwright-validation`), the end-to-end orchestrator
(`e2e-validate`), and the shipping-decision checklist
(`production-readiness-audit`).

The other **31 skills are fully functional and discoverable by Claude
Code** — they are specialized helpers (iOS simulator control, CI design
token audits, responsive-matrix sweeps, etc.) whose `SKILL.md` in the
repo is the canonical reference. Each row above links to the exact
`skills/<name>/SKILL.md` file on GitHub.
