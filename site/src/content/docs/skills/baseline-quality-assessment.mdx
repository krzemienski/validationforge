description: >
  Use BEFORE making any significant code change — refactor, dependency upgrade, migration, bug fix, performance optimization — to capture an immutable 'before' snapshot. After the change, compare against this baseline to prove two things: (1) what you intended to improve actually improved, (2) what you didn't intend to touch didn't regress. Prevents the classic 'I fixed X but accidentally broke Y'...
