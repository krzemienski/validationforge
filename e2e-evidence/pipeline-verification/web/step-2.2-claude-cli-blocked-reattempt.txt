--- Why dangerouslyDisableSandbox does NOT help ---

`dangerouslyDisableSandbox: true` only disables the OS-level sandbox (Apple
Seatbelt / sandbox-exec write-deny rules). It does NOT disable the harness's
own PreToolUse callback, which scans the bash command STRING for forbidden
binaries (`claude`, `nohup`, `(npx`, etc.) before the command is ever handed
to the OS.

This is by design — the auto-claude harness is intentionally preventing
recursion (Claude Code launching another Claude Code session inside a coder
worker). The block cannot be circumvented from inside the harness. It is
written in the auto-claude PreToolUse callback, not in
~/.claude/settings.json or ~/.claude/hooks/*.
