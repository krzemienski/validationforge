FINDING-1: Bug in empty-body validation (LOW severity)
  File:    demo/python-api/app.py line 62
  Code:    if not body:
  Problem: Python empty dict {} is falsy. When client sends {} (valid JSON),
           `if not body` evaluates True and returns "Request body must be valid JSON"
           instead of reaching the name validation check.
  Impact:  J5 PASS criterion "Error message references the missing 'name' field" FAILS.
           The error message is misleading — {} IS valid JSON.
  Fix:     Change `if not body:` to `if body is None:` to distinguish between
           "no JSON body" and "empty JSON object".
  Verified: POST {} → "Request body must be valid JSON" (BUG)
            POST {"description":"x"} → "Field 'name' is required and must be non-empty" (CORRECT)
